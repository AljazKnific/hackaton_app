# 01 — System Map

How the whole brain connects. This is the master map: which folder does what, and how they chain to turn a founder's product into a narrated promo. Read this first.

**Role in the app:** the orchestration layer — the pipeline the generation prompt follows.

---

## The Pipeline

```
STAGE 1 · INTAKE
  user pastes product info

STAGE 2 · BUILD BRAIN
  ICPOffer-v2 → ICP · pains · desires · objections · sharpened offer
                (validation gate: product, audience, core result defined?)

STAGE 3 · GENERATE
  Ideas-v2                → promo angles  (Pain × Trigger × Reframe)
  Emotions-v2             → which emotional trigger to pull
  ViralHooks-v2           → the hook (0–3s)
  ICPOffer-v2/06          → the 30-sec beat structure
  WritingSystem-v2        → craft the copy (sentence, value, persuasion)
  CTA-v2                  → the closing ask (22–30s)
  Voice-v2                → tone (picked by product) + voiceover delivery
  Structure-Formatting-v2 → script layout + voiceover cues

STAGE 4 · PICK
  user chooses a script (of ~3)

STAGE 5 · VOICE
  ElevenLabs → MP3 voiceover

STAGE 6 · HANDOFF
  script + voiceover + caption tips → user's editor
```

---

## What Each Folder Contributes

| Folder | Contributes | Stage |
|---|---|---|
| **ICPOffer-v2** | who it's for, the pains/desires, the sharpened offer, the beat map | 2 + 3 |
| **Ideas-v2** | the promo angles to choose from | 3 |
| **Emotions-v2** | the emotional trigger palette (canonical) | 3 |
| **ViralHooks-v2** | the opening hook | 3 |
| **WritingSystem-v2** | sentence craft, value delivery, persuasion | 3 |
| **CTA-v2** | the single closing ask | 3 |
| **Voice-v2** | tone selection + spoken delivery | 3 |
| **Structure-Formatting-v2** | script layout, beat timings, voiceover cues | 3 |

## Dropped from the old Twitter SOP

The original SOP also routed through `XAlgo-v2` (X algorithm gate) and `AntiPatterns-v2` (tweet tone filter). Both are **Twitter-only and not part of the app brain** — the app doesn't optimize for the X algorithm. Honesty/quality checks now live inside each folder (validation gates, "honest only" rules).
