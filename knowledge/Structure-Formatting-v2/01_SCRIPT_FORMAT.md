# 01 — Script Format

How the app lays out the promo script it hands to the user — so it's easy to read, record, and drop into the ElevenLabs voiceover.

**Role in the app:** the output format of the Stage 3 script (and what Stage 5 reads aloud).

---

## The Layout

Each script is delivered as five labelled beats with timings, one line per breath:

```
[0–3s · HOOK]
You did the work weeks ago. So why aren't you paid?

[3–8s · PROBLEM]
Chasing invoices. Awkward follow-ups. Payments that just… vanish.

[8–15s · TURN]
InvoiceZen sends the invoice in seconds — and chases the payment for you.

[15–22s · PROOF]
Freelancers using it get paid eleven days faster.

[22–30s · CTA]
Send your first invoice free. Start now.
```

---

## Rules

- **One line = one breath.** Break the script into short spoken lines (5–12 words), never paragraphs.
- **Label every beat** with its time window so the founder can pace the recording.
- **Total length ≈ 55–75 words** for a natural 30-second read. Longer = rushed.
- **Numbers as words** ("eleven days," not "11d") so the voice model reads them right.
- **No visual formatting in the spoken lines** — no ALL-CAPS, arrows, or bullets; those aren't heard (see `03` for on-screen text).

---

## Word-count guide (30 seconds)

| Beat | Seconds | ~Words |
|---|---|---|
| Hook | 0–3 | 6–10 |
| Problem | 3–8 | 10–15 |
| Turn | 8–15 | 14–20 |
| Proof | 15–22 | 10–15 |
| CTA | 22–30 | 8–14 |

Aim for the low end — spoken words take longer than they look.
