# 03 — On-Screen Text (optional)

Tips for the on-screen captions the user overlays when they add visuals. Optional — the app hands off the script + voiceover; the user builds the video. This is the one place the old visual-formatting wisdom still applies.

**Role in the app:** optional guidance included in the Stage 6 handoff.

---

## Core Rules

- **One idea per caption card.** Match the caption to the spoken line it's under.
- **Short and punchy** — a few words, not full sentences. The voice carries the detail; the text reinforces the point.
- **Hook first.** The opening caption should mirror the spoken hook so scrollers with sound off still stop.
- **One emphasis word** per card can be capitalized or colored — but only one. Four+ caps looks spammy.
- **Use a → for transformations** ("spreadsheet → paid in a day") — arrows read fast on screen.

---

## Caption skeleton (mirrors the beats)

| Beat | Caption idea |
|---|---|
| Hook | the pain, in 3–5 words ("still not paid?") |
| Problem | the symptom ("chasing invoices again") |
| Turn | the fix ("invoices that chase themselves") |
| Proof | the number ("paid 11 days faster") |
| CTA | the action ("start free →") |

---

## Note

Captions are visual, so unlike the spoken script they *can* use numerals ("11 days"), symbols, and a single emphasis. Keep them minimal — they support the voiceover, they don't repeat it word-for-word.
