# 06 — Content Angle Map

The master map that connects the brain to the output: **pain / desire / objection → video-script structure → CTA**. This is what Stage 3 uses to turn the brain into promo scripts.

**Feeds:** Stage 3 script generation directly. Replaces the old tweet-hook logic with 30-second video structures.

---

## Role in the app

- **Stage 3 (Generate):** for each pain/desire/objection in the brain, the app picks a **script structure** here and writes a 30-sec promo against it.
- Every structure is built on the same beat map as `05_OFFER_POSITIONING`: Hook → Problem → Turn → Proof → CTA.

---

## Video Script Structures (the promo templates)

Each is a 30-second shape. The app picks based on the customer's awareness stage and the angle.

| Structure | Shape | Best for | Awareness |
|---|---|---|---|
| **Pain-Twist-Fix** | Name the pain → make it worse → reveal the fix → CTA | Problem-aware customers | Problem Aware |
| **False Fix Callout** | "You think X solves it → here's why it doesn't → the real fix" | "I already have a way" objection | Solution Aware |
| **Cost-of-Inaction** | "Every [time] you do X, you lose [Y] → stop → CTA" | Price / "not broken" objections | Problem Aware |
| **Before/After** | Old messy way → new effortless way, side by side | Showing the transformation | Solution Aware |
| **The Math** | Say the numbers out loud → the value is undeniable → CTA | Price objection, ROI proof | Product Aware |
| **Straight Offer** | Promise → mechanism → proof → CTA | Ready buyers | Most Aware |

## The 30-Second Beat Map (every script obeys this)

| Time | Beat | Job | Source in the brain |
|---|---|---|---|
| 0–3s | **Hook** | Stop the scroll — name the pain or promise | `02_PAIN_POINTS` |
| 3–8s | **Problem** | Twist the knife — one vivid symptom | `02_PAIN_POINTS` |
| 8–15s | **Turn** | "Here's the fix / here's life instead" | `03_DESIRES` + mechanism |
| 15–22s | **Proof** | Number, before/after, or value math | `05` proof / value frame |
| 22–30s | **CTA** | One direct ask | `05` after-state |

---

## Voiceover Rules (applies to every generated script)

- Short spoken sentences — one idea per breath.
- Contractions and plain words; no jargon the ear can't parse.
- Numbers spoken cleanly ("sixty seconds," "eleven days").
- End on the CTA — never trail off.
- Read every script aloud (mentally) before finalizing: if a line can't be said in one breath, cut it.
