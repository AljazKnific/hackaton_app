# 07 — Spoken Cadence & Delivery

How the voice moves — rhythm, pacing, and how it's meant to be heard. This is what makes the ElevenLabs read sound natural instead of robotic.

**Role in the app:** shapes line breaks and pacing of the final script; feeds the voiceover.

---

## Core Cadence

| Element | Rule |
|---|---|
| Arc | Hook → Insight → Payoff — every promo follows it |
| Rhythm | one idea per line, one line per breath |
| Pacing | short lines build momentum; a beat of silence lands the turn |
| Flow | minimal clauses — no line should need a comma to survive |

---

## Pacing for Voiceover

- **Vary line length for rhythm.** A run of short lines, then one slightly longer, keeps it from sounding flat.
- **Use the pause.** The gap between the problem and the fix is where tension lands — write it as a line break, and let the voice breathe there.
- **Land the last line.** The CTA is the final beat — end on it cleanly, don't trail off.
- **Read it aloud (mentally) before finalizing.** If you stumble or run out of breath, rewrite the line.

---

## Plug-and-Play Script Shapes

These work with any tone mode and map onto the 30-sec beat structure:

```
[Pain] → [Twist] → [Fix] → [CTA]

[Before] → [After] → [How] → [CTA]

[Question they feel] → [Real answer] → [Proof] → [CTA]

[Bold outcome] → [Why it's possible] → [Proof] → [CTA]
```

---

## Rule

The script is a *spoken* thing first, a written thing second. If it reads fine but sounds clunky aloud, it's wrong. Cadence beats cleverness.
