# 03 — Value Delivery

Deliver a shift in perspective, not a lecture. By the end of the promo the viewer should think "that makes sense" or "I never saw it that way" — and feel smarter, not talked down to.

**Role in the app:** shapes the Turn beat (8–15s) and the overall payoff of the script.

---

## Core Principles

| Principle | Detail |
|---|---|
| One core idea | the whole promo supports a single point |
| Bite-size clarity | break the idea into simple, concrete statements |
| Recognition first | the viewer feels seen in the problem before you offer the fix |
| Don't over-explain | imply depth; let them connect the last dot themselves |
| Belief shift | value = making them see the old problem in a new way |

---

## Types of Value (pick one per promo)

| Type | What it does |
|---|---|
| Conceptual clarity | makes a principle "click" in one line |
| Practical fix | shows the concrete thing that solves it |
| Psychological shift | reframes how they think about the problem |
| Proof of principle | shows a result, then the lesson behind it |

---

## Structure Patterns (for the middle of the script)

**Hidden-cause:**
1. state the outcome they want
2. reveal what's really blocking it
3. deliver the fix

**Before → after:**
1. the old, painful way
2. the new way
3. the line that makes the shift land

**Result → lesson:**
1. a short, real result
2. the principle it proves
3. the distilled takeaway

---

## Neutral Examples (InvoiceZen)

- "People think getting paid faster is about chasing harder. It's about *when* the invoice lands."
- "You don't need better clients. You need invoices that follow up for you."

---

## Rule

Deliver one clear truth, make it feel earned, and leave the viewer feeling capable — never ignorant. If the script lectures, cut it back to the single idea.
